### 1.简述云计算的目标和主要特点

云计算的目标：

从**计算模式演变**的角度来说，是：**为计算工作提供更方便快捷的使用方式和更强大的计算能力**

从**信息处理模式**的角度来说，是：**提供在线获取和在线反馈的紧耦合，人机物一体的信息处理模式**

从**社会服务功能**的角度来说，是：**把计算能力变成像水电等共用服务一样，随用随取，按需使用。即实现对资源的有效集中管理，便于共享**

云计算的主要特点：

**超大规模**：“无限”多的计算资源；集成了计算技术，具有很强的技术性、工程性特点；强大的计算能力。

**虚拟化**：对于使用者而言不需要固定占有物理设备，按需分配、弹性伸缩、取用方便，突破时间、空间的界限。

**高可靠性**：服务器单节点出现故障可以通过虚拟化技术将分布在不同物理服务器上面的应用进行恢复或利用动态扩展功能部署新的服务器进行计算。

**通用性**：透明的云端计算服务，适用于极多种应用场景和需求。

**高可伸缩性**：应用部署快速而容易、软件/应用功能更新方便快捷。

**按需服务**：能够根据用户的需求快速配备计算能力及资源。

**极其廉价**：资源共享，成本低廉，降低企业IT基础设施建设维护费用；节省能源，绿色环保。

### 2.云计算按照服务层面分类，可以分为哪三类？简述每一类的作用和特点。

可以分为三类，基础设施即服务（IaaS）、平台即服务（PaaS）、软件即服务（SaaS）。

**基础设施即服务（IaaS）**：体现为云计算应用服务软件。将硬件设备等基础资源封装成服务提供给用户使用。通用性最强。案例：Amazon EC2/S3

特点：自助式、虚拟化的租赁服务，可以按等级、按需求弹性地索取服务。

**平台即服务（PaaS）**：体现为云计算软件支撑平台。对资源进行进一步抽象，为用户的应用程序提供运行环境。通用性与专用性居中。案例：Google App Engine，Microsoft Windows Azure

特点：由专门的平台服务提供商搭建和运营基础平台，并将该平台以服务的方式提供给应用服务运营商，为应用开发商提供敏捷性和快速响应需求的能力，节省应用开发时间和迁移成本。

**软件即服务（SaaS）**：体现为云计算硬件平台。将某些特定的应用软件功能封装为服务，具有强针对性。专用性最强。案例：Salesforce online CRM

特点：可扩展、高并发、高可复用性（多重租赁），以一套标准软件系统为多名客户同时提供服务，用户只需要按需使用，不用考虑开发维护问题。

### 3.简述怎样才算是云计算系统？

需要具备以下两个重要特征，其中第一个尤其重要：

1. **资源虚拟化和弹性调度解决小粒度应用资源共享**

   以按需分配的方式，将资源共享给小规模计算的大量用户，实现云计算的初衷。

2. **大数据储存处理和并行计算服务提供大粒度应用计算能力**

   为大规模计算需求端提供相较于独立平台算力强大而价格低廉的计算能力，满足其高性能、高强度计算的需求。如海量数据存储与处理，大规模并行计算等。

### 4.简述云原生系统需要包含的属性。

**容器化封装**：以容器为基础，提高整体开发水平，形成代码和组件复用，简化云原生应用程序的维护。在容器中运行应用程序和进程，并作为应用程序部署的独立单元，实现高水平资源隔离。

**自动化管理**：统一调度和管理中心，从根本上提高系统和资源利用率，同时降低运维成本。

**面向微服务**：通过松耦合方式，提升应用程序的整体敏捷性和可维护性。
